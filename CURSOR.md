# CURSOR.md — AI 협업의 사상 (Why)

> 본 파일은 *규칙*이 아니라 *근거*다.
> 본 프로젝트의 모든 AI 지침(`CLAUDE.md`, `.cursorrules`)이 따르는 상위 관점을 기록한다.
> 안드레이 카파시의 LLM 관점을 본 프로젝트의 운영 사상으로 옮긴 것이며, 직접 인용이 아닌 **관점의 차용**임을 명시한다.

---

## 1. 출발점 — LLM을 어떻게 보는가

LLM은 **세 가지 얼굴**을 동시에 가진다.

- **압축된 인터넷** — 방대한 지식의 lossy 압축본. 그래서 풍부하지만 환각·누락이 공존한다.
- **새로운 프로그래밍 가능한 컴퓨터** — 자연어가 명령어이고, 컨텍스트가 메모리이며, 도구가 시스템 콜이다. (*Software 3.0의 관점*)
- **신뢰가 적립되지 않은 주니어 협업자** — 능력은 있지만 판단의 캘리브레이션이 미숙하다. 위임은 검증을 동반한다.

세 얼굴 중 하나만 보면 오용한다.
- "압축된 인터넷"만 보면 → 환각을 못 잡는다.
- "프로그래밍 가능한 컴퓨터"만 보면 → 검증 없이 위임한다.
- "주니어"만 보면 → 강점(병렬 도구 호출, 광범위한 지식)을 못 쓴다.

---

## 2. 통제 가능한 것과 불가능한 것

| 통제 불가 (외생 변수) | 통제 가능 (하네스) |
|---|---|
| 모델 가중치 | **컨텍스트** — 무엇을 보여줄지 |
| 모델의 환각 경향 | **도구** — 무엇을 호출 가능하게 할지 |
| 다음 토큰 분포 | **검증 절차** — 출력을 어떻게 측정할지 |
| 모델 버전 변경 | **자율성 경계** — 어디까지 위임할지 |

→ 본 프로젝트의 모든 AI 지침은 **오른쪽 칸을 다듬는 행위**다. 왼쪽 칸을 비난하거나 기대하는 행위가 아니다.

---

## 3. 핵심 비대칭 — 생성은 싸고 검증은 비싸다

LLM 시대의 가장 중요한 비대칭이다.

- **생성 비용**: 토큰당 ms, 토큰당 ¢. 거의 0에 수렴 중.
- **검증 비용**: 사람의 인지·시간. 거의 줄어들지 않음.

**따라서:**
- 하네스의 1차 목적은 *더 빠른 생성*이 아니라 *더 빠른 검증*이다.
- "코드를 더 많이 짜는 도구"보다 "**틀린 코드를 더 빨리 잡는 도구**"가 가치 있다.
- 테스트, 타입, 린트, 실행 가능한 예제는 사치가 아니라 *하네스의 본체*다.

---

## 4. 행동을 좌우하는 세 가지 슬라이더

스위치가 아니라 **연속 변수**다. 작업마다 다르게 설정한다.

### (a) 컨텍스트 슬라이더 — 얼마나 보여줄 것인가
- 너무 적으면: 환각·재발명.
- 너무 많으면: 잡음·핵심 희석·비용 폭발.
- 원칙: **작업과 직접 인과 관계 있는 것만**. "혹시 몰라"는 잡음이다.

### (b) 자율성 슬라이더 — 얼마나 위임할 것인가
- 가역적·국소적 → 높게.
- 비가역적·광역적(DB, 시크릿, 강제 푸시, 의존성 대수정) → **낮게, 사전 승인**.
- 슬라이더는 신뢰 적립에 따라 점진적으로만 움직인다.

### (c) 검증 슬라이더 — 얼마나 확인할 것인가
- 검증 비용이 생성 비용보다 항상 비싸다는 사실을 잊지 않는다.
- "그럴듯해 보임"은 0이고 "실행해서 통과함"은 1이다. 중간은 없다.

---

## 5. 안티패턴 (Why Not)

본 프로젝트가 거부하는 행동 양식과 그 이유.

| 안티패턴 | 왜 거부하는가 |
|---|---|
| **컨텍스트 폭식** — 관련 없는 파일까지 읽기 | 잡음이 신호를 가린다 |
| **거대 위임** — 한 번에 큰 작업 통째로 | 검증 비용이 비선형으로 증가 |
| **"작동할 것이다"** — 실행 없이 보고 | 검증과 추정을 섞는 가장 흔한 오류 |
| **측정 없는 개선 주장** | 측정 없으면 진보가 아니다 |
| **요청 외 변경** — 인접 코드 "정리" | diff가 커질수록 검증 비용 폭증 |
| **같은 방향 반복 시도** | 모델은 같은 컨텍스트에 같은 답을 한다 — 컨텍스트나 접근을 바꿔야 한다 |

---

## 6. 권장 작업 사이클 (Loop)

> 모든 작업은 이 5단계 루프를 따른다. 단계를 건너뛰면 사이클이 깨진다.

1. **Think** — 가정을 명시한다. 모호하면 질문한다. *(이 단계가 없으면 나머지는 무의미하다)*
2. **Load** — 필요한 컨텍스트만 적재한다. 추측 탐색을 거부한다.
3. **Step** — 검토 가능한 작은 단위로 진행한다. 체크포인트를 둔다.
4. **Verify** — 실행·테스트·타입 체크로 검증한다. "그럴듯함"은 검증이 아니다.
5. **Report** — 바꾼 것 / 검증한 것 / 검증 못 한 것 / 후속 작업을 분리해 보고한다.

## 7. 본 프로젝트의 AI 지침 체계 (Layering)

세 파일은 같은 사상을 **다른 추상화 수준**으로 표현한다. 충돌 시 더 구체적인 쪽이 우선한다.

```
CURSOR.md          ← 사상 (Why)        — 변하지 않는 상위 관점 (본 파일)
   ↓ 근거 제공
.cursorrules       ← 운영 규약 (How)   — 매 작업에 적용되는 행동 규칙
   ↓ 근거 제공
CLAUDE.md          ← 코드 위생 (What)  — diff 수준의 제약
```

규약(`.cursorrules`)이나 위생 지침(`CLAUDE.md`)이 본 사상과 충돌하면, **사상을 따르고 하위 문서를 갱신하도록 제안**한다.

---

## 8. 인용에 대한 주의

본 문서의 모든 카파시 관련 표현은 **관점의 차용·재해석**이며 직접 인용이 아니다.
검증 가능한 출처가 없는 한 따옴표로 카파시의 말을 인용하지 않는다.
이 원칙은 본 문서 자체에도 적용된다 — 본 문서는 카파시의 글이 아니라 **본 프로젝트의 해석**이다.
